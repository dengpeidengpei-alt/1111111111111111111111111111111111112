## 本轮新增经验

### 1. 规则违规快速修复流程
基于今天的3个规则违规经验，总结出快速修复流程：
1. **停止**：立即停止当前违规操作
2. **记录**：立即写入ERROR.md（时间+违规行为+根因）
3. **修复**：更新相关规则文件
4. **同步**：更新LEARNINGS.md（新增条目）
5. **反思**：确认不会再犯

### 2. 并行任务分解模式
大任务分解为并行子任务的最佳实践：
1. 识别可并行部分（独立子任务）
2. 主线程负责核心流程
3. 后台Agent负责独立验证/补充
4. 每10分钟检查一次进度
5. 最多3个后台Agent同时运行

### 3. 自检增强版
在原5条自检基础上增加：
- 第⑤条：操作范围在E:/Claude内
- 并行任务判断规则

### 4. 长任务穿插判断优先级
- 紧急（用户明确等待）→ 暂停长任务，处理紧急
- 非紧急 → 记录待回复，继续长任务
