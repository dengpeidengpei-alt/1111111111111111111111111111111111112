# 自改进触发器

> 检测用户指令中的自改进意图，自动触发代码化流程

## 触发规则

| 触发词 | 模式 | 代码动作 |
|--------|------|----------|
| 持续工作 | 用户说"去睡觉/睡懒觉"后接"持续工作" | 启动queue_runner.py后台运行 |
| 没说不动 | 用户说"不要只说不做" | 立即执行当前任务，不询问 |
| 不限额度 | 用户说"额度用完" | 立即启动多模态学习循环 |

## 检测关键词

```
持续工作|不间断|继续干活|别停
去睡觉|去睡了|我走了|你继续
不要找我了|不用管我
```

## 执行流程

1. 检测到触发词
2. 识别用户意图（持续工作/不再干预）
3. 立即启动后台任务
4. 记录到 events/

## 自检清单

当用户说以下内容时，必须立即执行对应动作：

| 用户说 | 执行 |
|--------|------|
| "持续工作" | 启动queue_runner.py，不等待确认 |
| "去睡觉" | 启动夜间任务，开始OCR处理 |
| "不要找我" | 所有任务后台运行，不输出到用户 |

# 自动记忆触发器 (Auto Memory Trigger)

> 事件驱动型跨Session记忆系统

## 触发事件

| 事件 | 触发时机 | 存储位置 | Python调用 |
|------|----------|----------|------------|
| `session_end` | Session结束时 | LEARNINGS.md | `auto_memory_trigger.py --event session_end` |
| `rule_violation` | 规则违规时 | ERRORS.md + LEARNINGS.md | `auto_memory_trigger.py --event rule_violation --detail "类型\|详情"` |
| `user_feedback` | 用户反馈时 | FEEDBACK.md + LEARNINGS.md | `auto_memory_trigger.py --event user_feedback --detail "类型\|详情"` |
| `task_complete` | 任务完成时 | accomplishments/ | `auto_memory_trigger.py --event task_complete --detail "任务名\|详情"` |
| `knowledge_new` | 发现新知识时 | LEARNINGS.md | `auto_memory_trigger.py --event knowledge_new --detail "类型\|标题\|详情"` |

## 触发检测关键词

```
# session_end
再见|我走了|去睡觉|持续工作|你继续

# rule_violation
违规|违反|错了|不应该

# user_feedback
满意|不满意|很好|不行|继续

# task_complete
完成|搞定了|结束了|好了

# knowledge_new
发现|学到了|新知识|这个是
```

## 触发器实现

在 `auto_memory_trigger.py` 中实现，调用示例：

```bash
# Session结束
python auto_memory_trigger.py --event session_end --detail "完成12轮迭代"

# 规则违规
python auto_memory_trigger.py --event rule_violation --detail "操作范围|尝试读取C盘"

# 用户反馈
python auto_memory_trigger.py --event user_feedback --detail "表扬|用户表示满意"

# 任务完成
python auto_memory_trigger.py --event task_complete --detail "Wiki深度化|完成ML等5个条目"

# 发现新知识
python auto_memory_trigger.py --event knowledge_new --detail "方法论|RuFlo自动记忆|跨Session记忆机制"
```

## 启动时召回流程

每次启动Claude时，自动执行：

1. 读取 `accomplishments/INDEX.md` 获取近期成果
2. 读取 `FEEDBACK.md` 获取用户反馈
3. 读取 `LEARNINGS.md` 末尾3条 pending 学习
4. 推送到当前session上下文

## 启动时召回 (memory_recall.py)

> 每次启动Claude时自动执行，将跨Session记忆推送到上下文

### 使用方法

```bash
# 简洁报告（人类可读）
python .learnings/memory_recall.py --brief

# JSON输出（程序处理）
python .learnings/memory_recall.py --json
```

### 输出内容

1. **CURRENT** - 当前工作计划（从MEMORY.md）
2. **ACCOMPLISHMENTS** - 近期成果（从accomplishments/INDEX.md）
3. **FEEDBACK** - 用户反馈（从FEEDBACK.md）
4. **LEARNINGS** - 待处理学习（从LEARNINGS.md末尾3条pending）
5. **SIGNALS** - 待处理信号（从signals/目录）

### 验证结果

| 功能 | 状态 |
|------|------|
| 当前工作计划 | ✅ |
| 近期成果 | ✅ |
| 用户反馈 | ✅ |
| 待处理学习 | ✅ (3条) |
| 待处理信号 | ✅ |
