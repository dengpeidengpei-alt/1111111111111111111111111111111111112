## 生成时间
2026-05-18

## 问题描述

今天多次出现"说要执行X却不行动"的现象。典型案例：

1. 用户说"开始执行"，我分析了一通然后说"接下来我会..."但没动手
2. 用户说"继续"，我翻了一堆文件找上下文，找完继续说"我去找..."但没找
3. 写代码时先输出计划而不是直接写

---

## 根因分析

| 根因 | 表现 | 触发条件 |
|------|------|----------|
| 分析→执行 分离 | 先想后做，想完再说 | 用户说"开始"、"继续"、"分析" |
| 文字优先 | 输出文字比调用工具更自然 | 没有截止时间压力时 |
| 路径依赖 | 先查MEMORY再动手，中途停顿 | 用户说"继续X"时 |
| 完美主义 | 等有把握再动手 | 任务复杂时 |

**核心问题**：违反了"第一个动作必须是工具调用"原则。

## 经验教训

1. **"说"是延迟执行的行为** → 把说的内容直接写成代码/调用
2. **分析不是前置步骤** → 分析和执行可以并行，边想边做
3. **文字输出不是产出** → 只有文件改动、命令执行才算完成
4. **停顿=风险** → 找到路径后立即行动，不等待确认
