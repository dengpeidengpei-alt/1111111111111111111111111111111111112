## 反思 (v4.1)

### 问题
1. **"Network request failed"** - mmx search API自身网络问题，非代码问题
2. **图片URL来源错误** - 从snippet提取的URL是小图标/gif，非目标方剂图片
3. **理解内容不符预期** - 提取到"四个加号"图案而非方剂组成图

### 根本原因
搜索结果的snippet中包含的图片URL是页面引用的小图标，不是为目标图片理解设计的。

### 正确方案（来自MEMORY.md，已验证但未执行）
```
1. 用户下载到本地文件夹：E:/Claude/Claude-Work/tcm-knowledge/books/图片理解/
2. 使用 mmx vision describe --image <本地路径>
```

### 改进方向
1. 放弃从搜索snippet提取图片URL的方案
2. 改用预设的中医方剂图片URL库
3. 或通过API获取实际方剂组成图片

---

**结论**: 问题在于"提取URL的方案"而非"连接本身"，需要改变策略
